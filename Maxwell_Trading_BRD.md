# Business Requirements Document (BRD)
## Maxwell Trading Organisation — Stock Management Web & Mobile Application

---

## 1. Project Overview

**Client:** Maxwell Trading Organisation
**Platform:** Web Application + Mobile Application
**Core Purpose:** Build a closed-loop Stock Management System that tracks fabric/textile stock from the moment it is received from supplying companies, through job-work processing at mills, packing/repacking, godown storage, and final dispatch via delivery challans — with full traceability at every stage via Item Code, Meter, Rate, and Packing ID.

**Business Domain:** Textile trading — stock is received in bulk (large meter lengths), sent out for job work (processing) to mills, received back, repacked into smaller pieces, stored in a godown, and dispatched against delivery challans.

---

## 2. Objective

Create a single source of truth for stock movement that:
- Prevents stock discrepancies across every stage of the supply chain
- Provides full traceability from raw stock-in to final dispatch, down to the individual packing unit
- Gives role-based access to Admin, Godown staff, and Floor Managers
- Works on both web (for admin/back-office use) and mobile (for floor/godown/warehouse use)

---

## 3. User Roles

| Role | Responsibilities |
|---|---|
| **Admin** | Creates Stock In entries, Stock Out (job work) entries, creates Delivery Challans, has full visibility across all ledgers and reports |
| **Godown Staff** | Receives packed stock into godown, manages godown inventory |
| **Floor Manager** | Receives Delivery Challans created by Admin, confirms receipt, adds stock into challan (Item Code, Meter, Packing ID), manages dispatch to delivery address |
| **Packing Staff** | Independent role. Creates Packing entries by splitting received stock into smaller pieces, generates Packing ID with barcode, and prints the packing sticker |

## 4a. Item Charter (Item Master) — Core Reference Database

**Why:** Staff currently have to remember an item's quality, color, and other specs by memory, which causes confusion and mistakes. The **Item Charter** solves this by being the single master record for every item. Once set up, **every other module in the app (Stock In, Stock Out, Packing, Godown, Delivery Challan) refers to the item only by its Item Code** — the system pulls quality, color, and all other details automatically from the Item Charter wherever needed (screen display, printouts, reports), so staff never have to type or recall them manually.

| Field | Description |
|---|---|
| **Item Code** | Unique identifier — the *only* thing staff select/enter anywhere else in the app |
| **Item Name** | Descriptive name |
| **Quality** | Fabric quality/grade |
| **Color** | Color name/code |
| **Category** *(assumed)* | e.g., fabric type/segment — confirm if needed |
| **Unit of Measure** | Meter (default) |
| **HSN Code** *(assumed)* | If required for GST/invoicing — confirm |
| **Other Specs** *(assumed)* | Width, GSM, or any other textile-specific attributes — confirm which apply |
| **Created/Modified By & Date** | Audit fields |

**Key Rule:** Item Code is mandatory and unique. All screens across the app (Stock In, Job Work Out, Packing, Godown, Challan) should have an **Item Code search/select field** that auto-populates Quality, Color, and other details as read-only reference — never free-text entry of quality/color elsewhere in the system.

## 5. Workflow Stages (Stock Lifecycle)

### **Stage 1 — Stock In (Inward from Company)**
- Admin records stock received from supplying companies into the **In Ledger**.
- Captured fields: **Company Code, Item Code, Meter, Rate**
- Each entry generates a unique **Stock In ID / Lot Number** for traceability.
- System updates available stock quantity per Item Code.

### **Stage 2 — Stock Out (Job Work Dispatch to Mill)**
- Admin issues stock **out** for job work processing at a mill.
- Captured fields: **Item Code, Meter, Mill/Vendor, Job Work Type** *(assumed — confirm if rate/charges are tracked here too)*
- Stock is deducted from available inventory and marked as **"With Mill / In Job Work"** status — not yet fully out of the system, since it is expected to return.

### **Stage 3 — Stock In (Return from Mill / Delivery In)**
- Processed stock returns from the mill and is received back into the **In Ledger** again.
- Captured fields: Item Code, Meter (post-processing), reference to original Stock Out entry.

### **Stage 4 — Packing**
- Performed by the dedicated **Packing Staff** role.
- Returned stock is split into smaller pieces (e.g., a 100-meter piece becomes two pieces of 50m + 50m, or 60m + 40m, etc.).
- Each split/packing operation generates a unique **Packing ID**, which is:
  - Encoded as a **barcode** (linked to Item Code, Meter, and source lot)
  - Printed as a **sticker** at the time of packing, to be physically stuck on the piece
- Captured fields: Packing ID (auto-generated), Item Code (selected from Item Charter — Quality/Color auto-filled), Meter (per piece), reference to source Stock In lot.
- One original piece can generate **multiple Packing IDs** (one-to-many relationship).
- The barcode on the sticker allows the piece to be **scanned at every later stage** (Godown transfer, Delivery Challan dispatch) instead of manually entering Packing ID.

### **Stage 5 — Godown Transfer**
- Packed stock (identified by Packing ID) is moved and stored in the **Godown**.
- System tracks current location/status of every Packing ID: *In Transit / In Godown / Dispatched.*

### **Stage 6 — Delivery Challan & Dispatch**
- **Admin** creates a **Delivery Challan** specifying the delivery address.
- **Floor Manager** receives the challan and adds the actual stock being dispatched against it: **Item Code, Meter, Packing ID**.
- System deducts dispatched Packing IDs from godown stock and marks the challan as fulfilled.
- Challan should support partial fulfillment *(assumed — confirm if a challan can be dispatched in multiple parts)*.

## 6. End-to-End Stock Flow Diagram (Textual)

```
[Company] 
   → Stock IN (Company Code, Item Code, Meter, Rate)  → In Ledger
   → Stock OUT (Job Work to Mill: Item Code, Meter)    → Out Ledger / "With Mill" status
   → Stock IN (Return from Mill: Item Code, Meter)     → In Ledger (linked to original lot)
   → Packing (split into pieces, Packing ID generated) → Packing Register
   → Godown Transfer (Packing ID → Godown location)    → Godown Inventory
   → Delivery Challan (created by Admin)               → assigned to Floor Manager
   → Floor Manager confirms & adds stock (Item Code, Meter, Packing ID) → Dispatched
```

## 7. Functional Requirements

1. **Item Charter (Item Master)**
   - Central database of Item Code, Quality, Color, and all item specs
   - Every other module references items by **Item Code only**; Quality/Color auto-populate as read-only wherever the item appears (forms, printouts, reports)
2. **Ledger Management**
   - In Ledger (Company-wise, Item-wise) with Meter & Rate
   - Out Ledger (Mill/Job-work-wise) with Meter
   - Real-time stock balance per Item Code at every stage
3. **Packing Module**
   - Performed by Packing Staff role
   - Split a stock lot into multiple smaller pieces
   - Auto-generate unique Packing ID per piece
   - Generate **barcode** for each Packing ID and **print a sticker** at time of packing
   - Maintain parent-child traceability (original lot → packing pieces)
4. **Godown Module**
   - Receive stock into godown by **scanning the Packing ID barcode**
   - Track which Packing IDs are physically stored in which godown
   - Godown-wise stock report
5. **Delivery Challan Module**
   - Admin creates challan with delivery address
   - Floor Manager receives, confirms, and allocates stock against the challan by **scanning Packing ID barcode** (Item Code & Meter auto-filled)
   - Support challan status tracking (Created → Assigned → Dispatched → Delivered)
6. **Traceability & Reporting**
   - Full trace of any Packing ID back to its original Company Stock-In entry
   - Stock aging report, mill-wise pending job work report, godown stock report
7. **Role-Based Access Control**
   - Admin: full access, including Item Charter management
   - Packing Staff: packing entry, Packing ID/barcode generation & sticker printing only
   - Floor Manager: challan receipt & dispatch only
   - Godown Staff: godown receipt & transfer only

## 8. Non-Functional Requirements

- **Platforms:** Responsive Web App (Admin/back-office) + Mobile App (Floor Manager, Godown staff — Android priority *(confirm iOS need)*)
- **Offline support** for mobile users in godown areas with poor connectivity *(assumed — confirm if required)*
- **Barcode generation & printing:** system must generate a scannable barcode per Packing ID and support printing on sticker label printers (e.g., thermal label printers) at the packing station
- **Barcode scanning:** mobile app must support camera-based or hardware scanner-based scanning of Packing ID barcodes at Godown and Delivery Challan stages
- Audit trail/logs for every stock movement (who did what, when)
- Notifications to Floor Manager when a new Delivery Challan is assigned
